# ledgerproof-semantic-kernel

LedgerProof adapter for **Microsoft Semantic Kernel (Python)** — emits signed
[EU AI Act Article 50](https://artificialintelligenceact.eu/) transparency
receipts as a side channel.

- **Stream-aware** signing (no full-body buffering).
- **Side-channel** emission only (LLM and function results are never modified).
- **Local** verification (Ed25519 + canonical CBOR, no network round-trip).
- **Foundation-stewarded** open protocol.

## Quickstart — Kernel + Filter

```python
import asyncio
from semantic_kernel import Kernel
from semantic_kernel.functions import kernel_function

from semantic_kernel_ledgerproof import (
    LedgerProofFunctionFilter, LogEmitter, Ed25519Signer,
)

class WeatherPlugin:
    @kernel_function(description="Get the weather for a city.")
    def get_weather(self, city: str) -> str:
        return f"Sunny in {city}"

async def main():
    kernel = Kernel()
    kernel.add_plugin(WeatherPlugin(), plugin_name="weather")
    kernel.add_filter(
        "function_invocation",
        LedgerProofFunctionFilter(
            deployer_id="acme-deployer-001",
            emitter=LogEmitter("./receipts.jsonl"),
            signer=Ed25519Signer(),
        ),
    )
    result = await kernel.invoke(
        function_name="get_weather", plugin_name="weather", city="Paris"
    )
    print(result)

asyncio.run(main())
```

A receipt is appended to `./receipts.jsonl` for the function call.

## Direct chat-service usage

Direct calls on `OpenAIChatCompletion` / `AzureChatCompletion` bypass kernel
function filters. Wrap the service for chatbot receipts:

```python
from semantic_kernel.connectors.ai.open_ai import AzureChatCompletion
from semantic_kernel_ledgerproof import LedgerProofChatService

inner = AzureChatCompletion(
    deployment_name="gpt-4o", endpoint=..., api_key=...,
)
service = LedgerProofChatService(
    inner,
    deployer_id="acme-deployer-001",
    schema="azure_enterprise_session/v1",
    extra={
        "tenant_id": "00000000-0000-0000-0000-000000000000",
        "subscription_scope": "sub/00000000-0000-0000-0000-000000000000",
        "deployment_region": "westeurope",
    },
)
# Stream-aware: tokens are hashed as they arrive; nothing is buffered.
async for chunk in service.get_streaming_chat_message_content(
    chat_history=history, settings=settings
):
    print(chunk, end="")
```

## Receipt schemas

| Schema id | Article | Purpose |
| --- | --- | --- |
| `chatbot_session/v1` | 50(1) | User interacted with an AI system |
| `generated_content/v1` | 50(2) | Content was generated by AI |
| `agent_function_invocation/v1` | 50(1) variant | Kernel function called in an agentic flow |
| `azure_enterprise_session/v1` | 50(1) variant | Chatbot session bound to an Azure tenant / subscription scope |

All schemas hash the transcript (SHA-256) and record only opaque/pseudonymous
identifiers. **Argument values are never recorded**; only argument names.
GDPR-shaped validators reject email-shaped strings in identifier fields.

## Verification

Receipts are verifiable offline. The signed body is canonical CBOR; the
signature is Ed25519 over `SHA-256(canonical_CBOR(body))`. See the protocol
spec for the verifier algorithm — any Ed25519 + canonical-CBOR stack works.
