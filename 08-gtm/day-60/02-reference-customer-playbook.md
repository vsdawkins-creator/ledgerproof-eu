# Reference Customer Playbook

**Purpose:** Convert the first paying enterprise customers into named references that unlock the next ten. The first three references are disproportionately load-bearing — they are the answer to "who else in my sector has bought this?" which is the single most common question at the GC, CRO, and MLOps level.

**Owner:** Founder through Day 90; BD lead from Day 90 forward.

---

## The reference economy

| Reference tier | What customer commits to | What customer gets |
|---|---|---|
| **Logo** | Logo used on website + one-pager + investor materials | 10% credit on next annual contract; named in launch announcement |
| **Quote** | Logo + a single attributed quote | 15% credit; co-authored short blog post; named position on Foundation Advisory Council |
| **Case study** | Logo + quote + named case study with metrics + 15-min reference calls (max 4/year) | 20% credit; named Advisory Council seat; founder available for customer's own Board meeting (1x/year) |
| **Founder reference** | All of the above + willingness to be quoted publicly to press | 25% credit; pricing locked for 3 years; right of first review on protocol roadmap changes affecting customer's deployment |

The tiers are sequential — customers move up only after one quarter of production deployment under the previous tier. No tier is bought; all are earned by deployment evidence.

## The first three references — selection criteria

By Day 90, we need three reference customers with these properties:

1. **One Tier-1 EU bank, asset manager, or insurer.** This unlocks FSI sector pipeline. Target: a name from the [50 named target companies list]. Acceptable: any bank with €100B+ AUM operating in the EU.
2. **One Tier-1 EU media or content organization.** This unlocks the Article 50(2) synthetic media motion. Target: a major broadcaster, publisher, or studio.
3. **One Tier-1 EU enterprise outside FSI and media.** This proves the protocol generalizes. Target: industrial, pharma, telecom, or retail at €50B+ revenue.

Geography distribution target across the three: at least two EU jurisdictions, one of which is Germany or France.

## How to earn the first reference

The first reference customer will not become a reference until they trust the protocol survives a regulator examination. That means:

- **Pilot must succeed against documented SLO** (P99 ≤50ms, operator ≥99.5%, Stamp PDF delivered by Day 30)
- **Customer's first Audit-Ready Compliance Stamp PDF must pass internal audit review** at the customer (typically 2–4 weeks post-Day-30)
- **Customer must have at least one favorable interaction with a regulator** using the stamp — either an actual examination, an informational filing, or a Code of Practice consultation submission

This sequence takes 90–120 days from pilot start. The first reference therefore lands no earlier than Day 120 of the GTM plan. We will not pressure customers to become references before this sequence completes; doing so produces references the next buyer will not trust.

## Reference call playbook (15-minute format)

When a prospect requests a reference call, the playbook is:

| Time | Activity |
|---|---|
| 0–2 min | Introductions, reference confirms role and tenure |
| 2–7 min | Reference describes their original problem and what they evaluated |
| 7–11 min | Reference describes deployment experience and current operational state |
| 11–14 min | Prospect asks targeted questions |
| 14–15 min | Reference closes with one piece of unsolicited advice for the prospect |

The reference does not pitch. The reference does not handle objections. The reference does not commit on LedgerProof's behalf. We brief references explicitly: "your only job is to describe your own experience."

The founder or BD lead joins the first ten reference calls in listen-only mode to confirm the conversation matches what the reference agreed to. By the eleventh call we do not join.

## Reference fatigue management

Each named reference is capped at 4 reference calls per year. Once a reference hits the cap, prospects requesting that reference are routed to:

1. A different reference in the same sector if available
2. The Foundation Canonical Registry, where the reference customer's published Compliance Stamps are publicly inspectable
3. A 60-minute Shadow AI Inventory engagement, which substitutes for a reference call in many GC workflows

The 4-call cap is non-negotiable. Burning out the first three references costs the next thirty.

## Reference-to-prospect mapping

Sales prioritizes by sector match:

| Prospect sector | Preferred reference | Secondary reference |
|---|---|---|
| Tier-1 bank | The Tier-1 bank reference | Adjacent FSI reference |
| Asset manager | Bank reference | Bank reference |
| Insurer | Bank reference | Industrial reference |
| Media / broadcaster | Media reference | None — sector-specific |
| Pharma | Industrial reference | Bank reference |
| Industrial / manufacturing | Industrial reference | Bank reference |
| Telecom | Industrial reference | Media reference |
| Public sector | Foundation Canonical Registry only (no named references) | — |

Public sector procurement runs on different evidence rules; references from commercial peers are less valuable than published Foundation transparency reports. We do not put commercial references into public sector deals.

## When to decline a reference request

We decline reference requests when:

1. The prospect is a direct competitor to the reference customer
2. The prospect is a vendor evaluating LedgerProof for resale or white-label (those conversations go through partnership channels, not sales)
3. The prospect has not yet completed a 30-minute discovery call with sales (we do not burn reference time before the prospect is qualified)
4. The reference customer has already taken 4 calls this calendar year for any prospect

Declining is a positive signal. References that are easy to obtain are references the prospect will not trust.

## Public reference materials cadence

- **Quarterly:** One new customer case study (subject to availability)
- **Semi-annually:** One joint Foundation + customer published mapping (e.g., "How [Bank] implemented Article 50 receipts under DORA Article 28")
- **Annually:** One published reference customer panel at a major industry event (FSI ISAC, EU AI Act conference circuit, etc.)

Reference materials are reviewed by the customer's communications team before publication. Cycle time from draft to publication is 30–45 days.
