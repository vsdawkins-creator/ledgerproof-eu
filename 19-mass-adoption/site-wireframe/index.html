Open specification, no vendor lock-in

OCPP-AI is published under CC BY 4.0. The reference implementation is Apache 2.0. Twenty-nine framework adapters (LangChain, OpenAI SDK, Anthropic, Mistral, and more) ship today. Architecturally compatible with W3C Verifiable Credentials 2.0 and eIDAS 2.0.

Substrate-agnostic, EU-sovereign-ready

Receipt Merkle roots anchor to public, immutable substrates satisfying our published Anchor Interface. Current reference implementation: Bitcoin OP_RETURN. Primary alternative: European Blockchain Services Infrastructure (EBSI). Dual-anchor deployment recommended for EU-jurisdictional deployers.

GDPR-compatible by structure

Schema validation rejects personal data at the receipt layer. The 36-byte anchor payload carries no recoverable PII. Receipt-level Article 17 erasure is independent of the anchor. Verifier requires no SaaS dependency — only Bitcoin chain + public key + receipt.
